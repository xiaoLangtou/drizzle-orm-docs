# Drizzle Kit 迁移工具概述
<Prerequisites>
- 开始使用 Drizzle 和 `drizzle-kit` - [阅读这里](/docs/get-started)
- Drizzle 模式基础 - [阅读这里](/docs/sql-schema-declaration)
- 数据库连接基础 - [阅读这里](/docs/connect-overview)
- Drizzle 迁移基础 - [阅读这里](/docs/migrations)
</Prerequisites>

**Drizzle Kit** 是一个用于管理 SQL 数据库迁移的 CLI 工具。
<Npm>
  drizzle-kit
</Npm>
<Callout type="warning">
请确保先阅读 Drizzle 的[入门指南](/docs/get-started)和[迁移基础](/docs/migrations)，选择最适合您业务需求的 SQL 迁移流程。
</Callout>

基于您的模式，Drizzle Kit 允许您生成和运行 SQL 迁移文件，
直接将模式推送到数据库，从数据库拉取模式，启动 Drizzle Studio 以及一些实用命令。
<Npx>
drizzle-kit generate
drizzle-kit migrate
drizzle-kit push
drizzle-kit pull
drizzle-kit check
drizzle-kit up
drizzle-kit studio
</Npx>

| 命令 | 描述 |
| :--------------------------------------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| [`drizzle-kit generate`](/docs/drizzle-kit-generate) | 根据您的 Drizzle 模式生成 SQL 迁移文件，[详见](/docs/drizzle-kit-generate)。 |
| [`drizzle-kit migrate`](/docs/drizzle-kit-migrate) | 将生成的 SQL 迁移文件应用到数据库，[详见](/docs/drizzle-kit-migrate)。 |
| [`drizzle-kit pull`](/docs/drizzle-kit-pull) | 从数据库拉取(内省)模式并转换为 Drizzle 模式保存到代码库，[详见](/docs/drizzle-kit-pull) |
| [`drizzle-kit push`](/docs/drizzle-kit-push) | 将 Drizzle 模式直接推送到数据库，[详见](/docs/drizzle-kit-push) |
| [`drizzle-kit studio`](/docs/drizzle-kit-studio) | 连接到数据库并启动 Drizzle Studio 代理服务器，[详见](/docs/drizzle-kit-studio) |
| [`drizzle-kit check`](/docs/drizzle-kit-check) | 检查生成的迁移是否存在竞争条件(冲突)，[详见](/docs/drizzle-kit-check) |
| [`drizzle-kit up`](/docs/drizzle-kit-up) | 用于升级之前生成的迁移的快照，[详见](/docs/drizzle-kit-up) |
<br/>

Drizzle Kit 通过 [drizzle.config.ts](/docs/drizzle-config-file) 配置文件或 CLI 参数进行配置。<br/>
至少需要提供 SQL `dialect` 和 `schema` 路径以便 Drizzle Kit 知道如何生成迁移。
