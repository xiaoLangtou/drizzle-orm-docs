# 在现有项目中开始使用 Drizzle 和 Bun:SQLite

<Prerequisites>  
  - **dotenv** - 用于管理环境变量的包 - [点击了解更多](https://www.npmjs.com/package/dotenv)
  - **tsx** - 用于运行 TypeScript 文件的包 - [点击了解更多](https://tsx.is/)
  - **bun** - JavaScript 全能工具包 - [点击了解更多](https://bun.sh/)
  - **bun:sqlite** - 高性能 SQLite3 驱动的原生实现 - [点击了解更多](https://bun.sh/docs/api/sqlite)
</Prerequisites>

<FileStructure />

#### 步骤 1 - 安装所需的包

<Npm>
  drizzle-orm dotenv
  -D drizzle-kit tsx @types/bun
</Npm>

#### 步骤 2 - 设置连接变量

<SetupEnv env_variable='DB_FILE_NAME' />

<Callout type='info' title='重要'>
例如，如果您在项目根目录中有一个 SQLite 数据库文件，您可以使用这个示例：
```plaintext copy
DB_FILE_NAME=mydb.sqlite
```
</Callout>


#### 步骤 3 - 设置 Drizzle 配置文件

<SetupConfig dialect='sqlite' env_variable='DB_FILE_NAME'/>

#### 步骤 4 - 检查你的数据库

<IntrospectSqlite/>

#### 步骤 5 - 将代码转移到你的实际模式文件

<TransferCode/>

#### 步骤 6 - 将 Drizzle ORM 连接到数据库

<ConnectBun/>

#### 步骤 7 - 查询数据库

<QueryDatabase dialect='bun-sqlite' env_variable='DB_FILE_NAME' />

#### 步骤 8 - 运行 index.ts 文件

要使用 `bun` 运行脚本，请使用以下命令：
```bash copy
bun src/index.ts
```

#### 步骤 9 - 更新你的表模式（可选）

<UpdateSchema/>

#### 步骤 9 - 将更改应用到数据库（可选）

<ApplyChanges/>

#### 步骤 10 - 使用新字段查询数据库（可选）

<QueryDatabaseUpdated dialect='bun-sqlite' env_variable='DB_FILE_NAME' />
