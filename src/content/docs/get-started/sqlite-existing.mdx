# 在已有项目中使用 Drizzle 和 SQLite 入门指南

<Prerequisites>  
  - **dotenv** - 用于管理环境变量的包 - [了解更多](https://www.npmjs.com/package/dotenv)
  - **tsx** - 用于运行 TypeScript 文件的包 - [了解更多](https://tsx.is/)
  - **libsql** - SQLite 的一个分支，针对低查询延迟进行了优化，适用于全局应用程序 - [了解更多](https://docs.turso.tech/libsql)
</Prerequisites>

<InstallPackages lib='@libsql/client'/>

#### 步骤 2 - 设置连接变量

<SetupEnv env_variable='DB_FILE_NAME' />

<Callout type='info' title='重要提示'>
例如，如果你想在项目根目录下创建一个用于测试的 SQLite 数据库文件，你需要在实际文件名前加上 `file:`，因为这是 `LibSQL` 要求的格式，像这样：
```plaintext copy
DB_FILE_NAME=file:local.db
```
你可以查看 **[LibSQL 文档](https://docs.turso.tech/sdk/ts/reference#local-development)** 了解更多信息。
</Callout>
