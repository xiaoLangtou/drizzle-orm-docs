# Drizzle 和 Neon 入门指南

<Prerequisites>
  - **dotenv** - 用于管理环境变量的包 - [了解更多](https://www.npmjs.com/package/dotenv)
  - **tsx** - 用于运行 TypeScript 文件的包 - [了解更多](https://tsx.is/)
  - **Neon** - 无服务器 Postgres 平台 - [了解更多](https://neon.tech/docs/introduction)
</Prerequisites>

Drizzle 通过 `neon-http` 和 `neon-websockets` 驱动原生支持 Neon 连接。这些驱动在底层使用 **neon-serverless** 驱动。

使用 `neon-http` 和 `neon-websockets` 驱动，你可以通过 HTTP 或 WebSocket 而不是 TCP 从无服务器环境访问 Neon 数据库。对于单个、非交互式事务，通过 HTTP 查询更快。

如果你需要会话或交互式事务支持，或者需要完全兼容的 `pg` 驱动替代品，你可以使用基于 WebSocket 的 `neon-serverless` 驱动。你也可以使用 [Postgres](/docs/get-started/postgresql-new) 直接连接到 Neon 数据库。
